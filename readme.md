# 飛行日誌分析儀 (Flight Log Analyzer)

## 描述

一個用於分析飛行日誌 CSV 檔案的平台。它能夠將數據透過互動式圖表進行視覺化

## 主要功能

*   **日誌檔案解析**: 支援上傳並解析飛控產生的 `.csv` 格式日誌檔案。
*   **互動式數據圖表**:
    *   動態顯示多個飛行參數群組，如電源系統、飛行控制、撞擊分析等。
    *   所有圖表的時間軸同步，可透過主時間軸進行縮放與平移。
    *   支援平滑曲線顯示，提升圖表可讀性。
*   **飛行軌跡視覺化**:
    *   在 2D 地圖 (Leaflet) 上顯示飛行軌跡，並提供終點google map 。
    *   圖表與地圖時間同步
    *   播放, 地圖會顯示對應時間點的飛行器位置。
*   **報告匯出**: 可將圖表、軌跡地圖與筆記的完整分析頁面儲存為單一的 HTML 檔案，便於歸檔與分享。

## 使用流程

1.  **上傳日誌**: 點擊「上傳並分析日誌」按鈕，選擇您的 `.csv` 檔案。
2.  **等待分析**: 系統會自動解析檔案。完成後，您將在左側看到「日誌資訊」，右側看到「時間軸調整」與預設的圖表。
4.  **檢視圖表**: 在左側「日誌檢查功能」區塊，點擊按鈕（如「電源與電力系統」、「飛行控制系統」）來顯示或隱藏對應的圖表群組。
5.  **互動操作**:
    *   在「時間軸調整」區塊，拖曳選取框以縮放特定時間範圍。
6.  **記錄資訊**:
    *   在「筆記」區塊記錄您的分析發現。
7.  **儲存報告**: 填寫完所有資訊後，點擊「另存為 HTML 檔案」，即可將當前頁面完整保存下來。

## 本地端部署教學 (Local Deployment Tutorial)

這是一個純粹的前端應用程式，但依賴一個輕量級的 Node.js 開發伺服器來即時轉譯程式碼。

### 前置需求

*   一台電腦
*   現代網頁瀏覽器 (如 Chrome, Firefox, Edge)
*   Node.js (**建議使用 v20 LTS 或更新版本**)

### 步驟 1：取得專案檔案

將所有專案檔案（包含 `index.html`, `package.json`, `server.js` 等）下載並存放在您電腦上的一個資料夾中。

### 步驟 2：安裝伺服器工具

我們的開發伺服器需要一些工具 (Babel) 來即時轉譯 TypeScript 和 JSX 程式碼。

1.  **打開您的終端機** (Terminal) 或命令提示字元 (Command Prompt)。

2.  **切換到專案目錄**: 使用 `cd` 指令切換到您存放專案檔案的資料夾。
    ```bash
    cd path/to/your/project-folder
    ```

3.  **安裝依賴**: 執行以下指令。這個指令會讀取 `package.json` 檔案並自動下載所需的工具。**這個步驟只需要在第一次設定時執行**。
    ```bash
    npm install
    ```

### 步驟 3：啟動伺服器

1.  **在同一個終端機視窗中**，執行以下指令來啟動伺服器。
    ```bash
    npm start
    ```

2.  **開啟應用程式**: 終端機將會顯示伺服器正在運行的網址，通常是 `http://localhost:3000`。在您的網頁瀏覽器中打開此網址，即可開始使用本分析工具。
